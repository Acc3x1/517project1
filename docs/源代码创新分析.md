### 2.1 温度仿真算法（自研） - 15分

#### 传统方案问题
- 需要真实加热炉硬件
- 需要温度传感器设备
- 需要串口通信驱动
- 成本高，维护复杂

#### 本方案创新
```java
// SimulationEngine.java - 自研温度仿真引擎
public TemperatureData update(int currentTime) {
    // 升温阶段算法
    if (tf1 < targetTemp - stableThreshold) {
        double increment = heatingRate * 0.8; // 800ms间隔
        tf1 += increment + randomNoise();
        tf2 += increment + randomNoise();
        ts = tf1 * 0.3 + noiseSmall;  // 样品温度低值跟随
        tc = tf1 * 0.25 + noiseSmall;
    }
    // 稳定阶段算法
    else {
        tf1 = targetTemp + randomNoise(); // 钳位到750°C
        stableCount++;
    }
    // 记录阶段算法
    if (isRecording) {
        ts += (tf1 * 0.95 - ts) * 0.02; // 表面温指数跟随
        tc += (tf1 * 0.85 - tc) * 0.01; // 中心温更慢跟随
    }
    return new TemperatureData(time, tf1, tf2, ts, tc, tCal);
}
```

#### 技术突破
1. **物理模型简化**：基于热传导原理简化，升温线性+噪声，稳定钳位
2. **多通道独立噪声**：每个通道独立Random，模拟真实传感器差异
3. **阶段自适应**：根据状态自动切换算法，无需手动干预

#### 对比优势
| 指标 | 传统硬件方案 | 本仿真方案 |
|------|-------------|-----------|
| 硬件成本 | >10万元 | 0元 |
| 维护成本 | 年维护费>1万 | 0元 |
| 数据采集 | 需串口驱动 | 纯软件实现 |
| 可移植性 | Windows专属 | 跨平台 |

#### 传统方案问题

- 状态无保护，可能跳步
- 用户体验差，易出错

```java
// StateMachine.java - 5状态定义
public enum StateMachine {
    IDLE, PREPARING, READY, RECORDING, COMPLETE;

    // 状态转换规则
    public boolean canTransitionTo(StateMachine target) {
        switch (this) {
            case IDLE: return target == PREPARING;
            case PREPARING: return target == READY || target == IDLE;
            case READY: return target == RECORDING || target == PREPARING;
            case RECORDING: return target == COMPLETE;
            case COMPLETE: return target == PREPARING;
            default: return false;
        }
    }
}

// TestController.java - 状态流转控制
public boolean startRecording() {
    if (!currentState.canTransitionTo(StateMachine.RECORDING)) {
        logger.warn("当前状态 {} 无法开始记录", currentState);
        return false; // 状态保护，防止误操作
    }
    setState(StateMachine.RECORDING);
    simulationEngine.startRecording();
    sendMessage("开始记录，计时开始");
    return true;
}
```

#### 技术突破
1. **状态转换规则硬约束**：`canTransitionTo()`方法防止非法状态跳转
2. **自动状态检测**：温度稳定自动从Preparing切换到Ready
3. **回调通知机制**：状态变化触发UI更新，解耦设计

### 2.3 数据采集定时机制 - 10分

#### 传统方案问题
- 串口通信复杂，需要驱动
- Windows平台依赖
- 硬件兼容性问题

#### 本方案创新
```java
// DaqWorker.java - 800ms定时数据采集
public class DaqWorker {
    private final int intervalMs = 800; // 800ms间隔

    public void start() {
        scheduler = Executors.newSingleThreadScheduledExecutor();
        scheduler.scheduleAtFixedRate(() -> {
            doWork(); // 每800ms执行数据采集
        }, 0, intervalMs, TimeUnit.MILLISECONDS);
    }

    private void doWork() {
        TemperatureData data = engine.update(currentTime);

        // 状态检查
        if (controller.getCurrentState() == PREPARING) {
            controller.checkReady(); // 自动检测Ready状态
        }

        // 线程安全回调
        Platform.runLater(() -> updateUI(data));
    }
}
```

#### 技术突破
1. **仿真间隔匹配真实采集**：800ms模拟真实Modbus采样周期
2. **ScheduledExecutorService**：后台线程定时，不阻塞UI
3. **Platform.runLater()**：解决JavaFX线程安全问题
